# План реализации личного кабинета и multi-company платформы

- [x] Зафиксировать модель владения: пользователь существует один раз и может состоять в нескольких компаниях.
- [x] Использовать компанию как tenant: должности, оценки, формы и файлы всегда принадлежат одной компании.
- [x] Проверять членство пользователя в компании на backend для каждого закрытого запроса; `company_id` от клиента не считается доверенным.

## Критерии готовности

1. Новый пользователь не попадает в рабочий интерфейс без созданной компании.
2. Один пользователь создаёт несколько компаний и переключается между ними из меню профиля.
3. Должности, оценки, формы и уведомления одной компании не видны в другой.
4. Прямой запрос UUID чужой должности возвращает `404`, не раскрывая её существование.
5. Пароли и исходные session-токены отсутствуют в БД.
6. Существующие сценарии оценки Hay продолжают работать и проходят автоматические тесты.
